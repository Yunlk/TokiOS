## 宣言

**操作系统没那么可怕。**

这堆东西说到底就是几个概念：中断、分页、系统调用。每个都能用一页纸讲清楚。
一个电子信息的大一学生，啃两周概念，也能让 CPU 乖乖听自己的话。

这里没有魔法。只有 C、汇编、和一份 `intel-sdm.pdf`。

## 这是什么

TokiOS — 纯 C + AT&T 汇编，从 bootloader 到 shell，全部手写。零依赖。零借口。

```
TokiOS> cout hello world
hello world

TokiOS> where
row=3, col=0
```

**Shell 设计哲学**：新手敲第一个命令就能看懂。想深挖的人，每个命令下面都是完整的内核机制。自然语言动词，无脑入门，天花板在文件系统驱动之后。

## 构建

32 位 x86 工具链。四行，一把梭。

```bash
as --32 boot.S -o boot.o
gcc -m32 -c *.c -ffreestanding -nostdlib -fno-pie -fno-stack-protector
ld -m elf_i386 -T linker.ld *.o -o tokios.bin
```

没有 cmake。没有 autotools。没有 `package.json`。

## 运行

```bash
qemu-system-i386 -machine pc -kernel tokios.bin
```

QEMU 7 到 11。Multiboot、PVH，你来哪个它接哪个。

## 里面有什么

**`boot.S`**
CPU 醒来的第一口气。Multiboot 头跟 QEMU 对暗号，GDT 铺好平坦内存，IDT 给中断画了门牌号，IRQ 存根在门口等着。这里没什么"启动流程"——就是顺序往下跑的汇编。

**`kernel.c`**
整个内核唯一一个函数。清屏、打个招呼、交棒给 shell。三行逻辑，剩下的都是态度。

**`gdt.c` / `gdt.h`**
段描述符表。Code 段 + Data 段，Ring 0 最高权限。没有用户态，所以也没有保护——自己写的内核怕什么。写它就是走一遍 Intel 手册 3.4.5 节的翻译题。

**`idt.c` / `idt.h`**
中断向量表。PIC 重映射让 IRQ0 不再跟 CPU 异常撞车。异常？打一行字告诉你几号错了，然后继续。优雅，但不浪费代码。

**`isr.c` / `isr.h`**
中断分派中心。只给键盘开了门，剩下的当不存在。没必要现在就写完所有驱动。

**`keyboard.c`**
128 项 scancode→ASCII 查表。回显到 VGA，光标跟着走，退格真的能退。一个文件搞定输入缓冲和回显——因为键盘本来就不该拆成三个文件。

**`shell.c` / `shell.h`**
`strncmp` + `atoi` + `write_int` + 命令分发。没有 lexer，没有 parser，没有 AST。就是 if-else。能跑的命令比能背的单词还少。

**`linker.ld`**
告诉链接器：Multiboot 头放最前面，不然 QEMU 不搭理你。`.text` 放代码，`.rodata` 放字面量，`.bss` 留给没初始化的全局变量。就是按地址分分类。

## 教条

- **不需要 `malloc`，因为你还没写。** 堆是特权，不是必需品。
- **中断是内核唯一的"多态"。** 唯一的异步，唯一的接口。不能再多了。
- **分页就是一张大数组查表。** 别看那些花里胡哨的术语，MMU 只是多了一层查表而已。
- **内核不 panic，它优雅停机。** 是的，物理意义的停机。你输 `shutdown`，它执行 `hlt`。
- **系统调用不超过一屏代码。** 如果有人写的 syscall 需要翻页，他在做应用层的事。
- **堆栈 4KB 够用。** 不够说明你在栈上干不该干的事。
- **启动信息的字数 ≤ 内核版本号位数。** "TokiOS booted."，六个字母，三个音节。够了。
- **注释解释"为什么"，代码解释"怎么做"。** 如果一段代码需要注释说"这是什么"，改名。
- **一个文件一个概念。** 不是一个函数一个文件。键盘驱动就该一个文件，它不配三个。
- **代码即文档。** ASCII 艺术是附赠的。README 也是。
